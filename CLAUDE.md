# AutoTex Book Generator

Agent-based scientific book generator in LaTeX. Claude Code is the primary agent — it plans, writes chapters, reviews formatting, and fixes issues. The Python package (`autotex/`) provides utility commands for compilation, image generation (Gemini), and PDF rendering.

## Project Structure

```
config.yaml          - Book metadata, KDP settings, Gemini config
main.tex             - Root LaTeX document (generated by `autotex update-main`)
autotex-book.sty     - Custom style: colored boxes, KDP page sizes, scientific packages
text/chapterNN.tex   - Individual chapter files (you write these)
assets/              - Generated images (created by `autotex images`)
build/               - Compilation output (PDF, aux files, rendered PNGs)
autotex/             - Python utility package
```

## CLI Commands

All commands: `python -m autotex <command>`

| Command | Description |
|---------|-------------|
| `init <name>` | Create a new book project in `books/<name>/` |
| `compile` | Compile main.tex to PDF via pdflatex (2 passes) |
| `render <pages...>` | Render PDF pages to PNG (0-based page numbers) |
| `render-chapter <num> <title>` | Render last N pages of a specific chapter |
| `images <file.tex>` | Extract image markers, generate via Gemini, replace with \includegraphics |
| `search <text>` | Find pages containing text in the compiled PDF |
| `update-main` | Regenerate main.tex from config.yaml chapters and KDP settings |
| `info` | Print current project configuration |

For book-specific commands, use `--config` to target a book subfolder:
```
python -m autotex --config books/<name>/config.yaml <command>
```

## Book Subfolder Workflow

Each book lives in its own self-contained subfolder under `books/`. The root project is the engine/template — never modify root files when working on a book.

### Creating a new book
```bash
python -m autotex init my-book
```
This creates `books/my-book/` with `config.yaml`, `CLAUDE.md`, `text/`, `assets/`, `build/`.

### Working on a book
All CLI commands use `--config` to target the book:
```bash
python -m autotex --config books/my-book/config.yaml update-main
python -m autotex --config books/my-book/config.yaml compile
python -m autotex --config books/my-book/config.yaml render-chapter 1 "Introduction"
python -m autotex --config books/my-book/config.yaml images text/chapter01.tex
```

### Book folder structure
```
books/my-book/
├── config.yaml    - Book metadata, KDP settings (edit this for planning)
├── CLAUDE.md      - Book-specific instructions (topic, audience, style)
├── main.tex       - Generated by update-main
├── text/          - Chapter .tex files
├── assets/        - Generated images
└── build/         - Compilation output (PDF, PNGs)
```

### Key rules
- **Never edit root-level** `config.yaml`, `main.tex`, `text/`, or `assets/` when working on a book
- Edit the **book's** `CLAUDE.md` for book-specific behavior (topic, tone, audience)
- `autotex-book.sty` stays at the repo root — the compiler finds it automatically via TEXINPUTS

## Book Planning

When asked to plan a book:
1. Run `python -m autotex init <book-name>` if the book subfolder doesn't exist yet
2. Read the book's `config.yaml` for existing settings
3. Create a structured plan with chapters, each having: number, title, sections (3-6), description
4. Build a logical progression from fundamentals to advanced topics
5. Include an introductory chapter and a concluding chapter
6. Update the book's `config.yaml` — set `book.title` and `book.chapters` array
7. Run `python -m autotex --config books/<name>/config.yaml update-main` to regenerate main.tex

## Writing LaTeX Chapters

### Guidelines
- Use `\chapter{Title}` and `\label{ch:slug}` at the start
- Use `\section{}`, `\subsection{}`, `\subsubsection{}` for structure
- Always add `\label{}` to chapters, sections, figures, tables, and equations
- Use `\cref{}` for cross-references (e.g., `\cref{fig:example}`)
- Use booktabs commands (`\toprule`, `\midrule`, `\bottomrule`) for tables
- Use the custom colored boxes (definitionbox, theorembox, examplebox, notebox) to highlight key concepts
- For AI-generated images, use the `%%IMAGE_REQUEST{...}%%` marker
- Keep paragraphs focused and well-structured
- Include mathematical notation where appropriate
- Do NOT include `\documentclass`, `\usepackage`, `\begin{document}`, or `\end{document}` — only write the chapter content starting with `\chapter{}`

### Available Environments

1. **Definition box:**
   ```latex
   \begin{definitionbox}[Title]
     Content here
   \end{definitionbox}
   ```

2. **Theorem box:**
   ```latex
   \begin{theorembox}[Theorem Name]
     Content here
   \end{theorembox}
   ```

3. **Example box:**
   ```latex
   \begin{examplebox}[Example Title]
     Content here
   \end{examplebox}
   ```

4. **Note/Warning box:**
   ```latex
   \begin{notebox}[Important]
     Content here
   \end{notebox}
   ```

5. **Figures** (images are in the `assets/` directory):
   ```latex
   \begin{figure}[htbp]
     \centering
     \includegraphics[width=0.8\textwidth]{filename}
     \caption{Caption text.}
     \label{fig:label}
   \end{figure}
   ```

6. **Tables** (use booktabs):
   ```latex
   \begin{table}[htbp]
     \centering
     \begin{tabular}{lcc}
       \toprule
       Header 1 & Header 2 & Header 3 \\
       \midrule
       Data & Data & Data \\
       \bottomrule
     \end{tabular}
     \caption{Caption text.}
     \label{tab:label}
   \end{table}
   ```

7. **Code listings:**
   ```latex
   \begin{lstlisting}[language=Python, caption=Caption]
     code here
   \end{lstlisting}
   ```

8. **TikZ diagrams:**
   ```latex
   \begin{tikzpicture}
     ...
   \end{tikzpicture}
   ```

9. **PGFPlots graphs:**
   ```latex
   \begin{tikzpicture}
     \begin{axis}[xlabel=X, ylabel=Y, title=Title]
       \addplot coordinates {(0,0) (1,1) (2,4)};
     \end{axis}
   \end{tikzpicture}
   ```

10. **Math** (amsmath, amssymb, mathtools are loaded):
    ```latex
    Inline: $f(x) = x^2$
    Display: \[ \int_0^\infty e^{-x} \, dx = 1 \]
    Aligned:
    \begin{align}
      a &= b + c \\
      d &= e + f
    \end{align}
    ```

### Image Request Markers

To request a Gemini-generated image, place this marker on its own line:
```
%%IMAGE_REQUEST{description="A diagram showing ...", filename="chNN-descriptive-name", aspect_ratio="16:9"}%%
```
After running `python -m autotex images text/chapterNN.tex`, the marker is replaced with a `\begin{figure}...\includegraphics...\end{figure}` block.

## Chapter Generation Workflow

Set `CFG=books/<name>/config.yaml` for the active book. For each chapter:
1. Read the plan from the book's `config.yaml`
2. Read previous chapters (`books/<name>/text/chapter01.tex`, etc.) for context and continuity
3. Write the chapter content to `books/<name>/text/chapterNN.tex`
4. Run: `python -m autotex --config $CFG images text/chapterNN.tex` (generates images and replaces markers)
5. Run: `python -m autotex --config $CFG compile`
6. If compilation fails, read error output, fix the LaTeX, recompile (up to 3 attempts)
7. Run: `python -m autotex --config $CFG render-chapter N "Chapter Title"` to get PNGs
8. Read the PNG files to visually review formatting
9. If issues found, edit the LaTeX and repeat from step 5 (up to 3 attempts)
10. Move to next chapter

For very long books (15+ chapters), read only the most recent 2-3 chapters for context continuity rather than all of them.

## Visual Review Checklist

When reviewing rendered PDF pages, check for:
1. Text overflowing into margins
2. Bad figure or table placement (floating too far from reference)
3. Orphan lines (single line at top of page) or widow lines (single line at bottom)
4. Overlapping elements (text over figures, tables over margins)
5. Consistent styling (fonts, spacing, heading styles)
6. Proper margins and whitespace
7. Readability and visual balance
8. Missing or broken cross-references (shown as "??")
9. Overfull or underfull hbox warnings visible as text jutting out

## Config Format

```yaml
book:
  title: "Book Title"
  author: "Author Name"
  chapters:                    # Populated after planning
    - number: 1
      title: "Chapter Title"
      sections: ["Section 1", "Section 2"]
      description: "Brief description"

api:
  gemini_model: "gemini-3-pro-image-preview"
  # API key from environment: GOOGLE_API_KEY

kdp:
  enabled: false
  trim_size: "novel"           # pocket|digest|novel|royal|crown|large|letter|ustrade|uktrade
  bleed: false
  gutter: ""                   # override (e.g. "0.5in"), empty = auto 0.375in
  paper: "white"               # white|cream

latex:
  compiler: "pdflatex"
  compiler_args: ["-interaction=nonstopmode", "-halt-on-error"]
  root_file: "main.tex"
  output_dir: "build"

review:
  max_revision_iterations: 3
  review_dpi: 200
  pages_per_review: 2
  final_review_interval: 5
```

## Prerequisites

- Python 3.13+
- pdflatex (MiKTeX on Windows, TeX Live on Linux/macOS)
- `GOOGLE_API_KEY` environment variable (for image generation)
- `pip install -r requirements.txt`
